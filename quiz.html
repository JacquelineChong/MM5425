Python Basics & Data Types (Q1–Q12)
Q1What does 10 / 3 return in Python? What about 10 // 3?
Answer
10 / 3 returns 3.333... (a float). 10 // 3 returns 3 (integer/floor division — it rounds down).
Q2You try to run "Revenue: " + 5000 and get an error. Why, and how do you fix it?
Answer
Python can't concatenate a string with a number directly. Fix: "Revenue: " + str(5000).
Q3What is the difference between = and == in Python?
Answer
= is assignment (stores a value). == is comparison (checks if two values are equal). A very common beginner mistake.
Q4You have a list sales = [200, 180, 220]. How do you add a new week's sales of 210 to the end?
Answer
sales.append(210). The list becomes [200, 180, 220, 210].
Q5How do you get the last element of a list without knowing its length?
Answer
Use negative indexing: sales[-1]. In Python, -1 always refers to the last element.
Q6What does range(5) produce?
Answer
The numbers 0, 1, 2, 3, 4 (five values starting from 0). The end value 5 is excluded.
Q7What does "hello world".split() return?
Answer
['hello', 'world'] — it splits the string on whitespace into a list of words.
Q8What does int(3.7) return? Does it round?
Answer
It returns 3. It does not round — it truncates (cuts off the decimal part) toward zero.
Q9In an if statement, should you use and/or or &/|?
Answer
Use and/or in regular Python if/else statements. Use &/| only for element-wise pandas Series operations (and wrap each condition in parentheses).
Q10Why do we write functions with def? What's the benefit?
Answer
Functions let you package reusable code. Instead of copying the same block multiple times, you define it once and call it whenever needed. This reduces errors and makes code easier to maintain.
Q11What does type(42) return?
Answer
<class 'int'> — it tells you the variable is an integer.
Q12You write s = "Analytics". What does s[0:4] give you?
Answer
"Anal" — characters at positions 0, 1, 2, 3. The stop index (4) is excluded, just like list slicing.
Loading & Inspecting Data (Q13–Q25)
Q13Your manager emails you a file called customers.csv. What pandas function loads it?
Answer
pd.read_csv('customers.csv'). For Excel files (.xlsx), use pd.read_excel() instead.
Q14You loaded a dataset and want to quickly see the first few rows. What do you type?
Answer
df.head() — shows the first 5 rows by default. Use df.head(10) to see 10.
Q15You want to know how many rows and columns your dataset has. What do you use?
Answer
df.shape — returns something like (5000, 14) meaning 5000 rows and 14 columns. Note: no parentheses — it's an attribute, not a method.
Q16What is the difference between df.info() and df.describe()?
Answer
df.info() shows column names, data types, and non-null counts (good for spotting missing values). df.describe() shows numeric summary statistics like mean, std, min, max, and quartiles.
Q17You run df.describe() but don't see your Gender column. Why?
Answer
describe() only summarizes numeric columns by default. For text/object columns, use df[['Gender']].describe().
Q18How do you check how many missing values each column has?
Answer
df.isnull().sum() — shows the count of NaN values per column. isnull() and isna() are identical.
Q19What does df.columns return?
Answer
An Index object listing all column names. Use list(df.columns) if you need a plain Python list.
Q20Before building any model, what are the first things you should check about your dataset?
Answer
Use df.shape for dimensions, df.info() for types and missing values, df.describe() for numeric summaries, value_counts() for categorical distributions, and isnull().sum() for missing counts. Plot histograms to see distributions.
Q21You want to see how many unique product categories exist in your data. What do you use?
Answer
df['Product line'].nunique() for the count, or df['Product line'].unique() to see the actual values.
Q22What is the difference between value_counts() and nunique()?
Answer
value_counts() returns the count for each unique value (a full breakdown). nunique() returns just a single number — how many distinct values there are.
Q23You want to see the bottom 5 rows of your DataFrame. What do you use?
Answer
df.tail(5) or just df.tail() (default is 5).
Q24Your dataset has a column called Rating. You want to see the average, min, max, and quartiles for just that column. How?
Answer
df['Rating'].describe() — gives count, mean, std, min, 25%, 50% (median), 75%, and max.
Q25Why is it important to look at the median (50%) and not just the mean when exploring data?
Answer
The mean can be pulled by extreme values (outliers). The median shows the "typical" value — 50% of data is above it and 50% below. If mean and median are very different, the data is likely skewed.
Data Cleaning & Preprocessing (Q26–Q45)
Q26You notice that the Income column has 50 missing values. Your manager says "just fill them with something reasonable." What would you do?
Answer
Use df['Income'].fillna(df['Income'].mean(), inplace=True) to replace missing values with the column average. If the data is skewed, the median is a safer choice: df['Income'].fillna(df['Income'].median(), inplace=True).
Q27What is the difference between fillna() and dropna()?
Answer
fillna() replaces missing values with a value you choose (mean, median, zero, etc.). dropna() removes entire rows (or columns) that contain missing values. Use fillna when you want to keep all your data; use dropna when missing rows are few and not worth guessing.
Q28You want to remove the ID column because it has no analytical value. How do you drop it?
Answer
df = df.drop('ID', axis=1). Remember: axis=1 means columns, axis=0 means rows.
Q29A colleague says "I used fillna but the DataFrame still has NaN values." What did they probably forget?
Answer
They probably forgot inplace=True. Without it, fillna() returns a new DataFrame and the original stays unchanged. Either use inplace=True or reassign: df['col'] = df['col'].fillna(0).
Q30Your dataset has a Gender column with values "Male" and "Female". A machine learning model needs numbers. How do you convert it?
Answer
For a two-value column, use df['Gender'] = df['Gender'].map({'Male': 1, 'Female': 0}). For columns with many categories, use pd.get_dummies() to create one-hot encoded columns.
Q31What does one-hot encoding do, and when would you use it?
Answer
It converts a categorical column into multiple 0/1 columns — one per category. For example, a "Color" column with Red/Blue/Green becomes three columns: Color_Red, Color_Blue, Color_Green. Use it when a column has more than two categories and you need numeric input for a model.
Q32You run pd.get_dummies(df, columns=['City']) and now have 50 new columns. Is there a way to reduce that?
Answer
Use drop_first=True: pd.get_dummies(df, columns=['City'], drop_first=True). This drops one category per column (it can be inferred from the others), reducing columns by one and avoiding multicollinearity.
Q33You have a price column that should be numeric, but df.info() shows it as "object" (text). How do you fix it?
Answer
df['price'] = pd.to_numeric(df['price'], errors='coerce'). The errors='coerce' turns any unparseable text values into NaN instead of crashing.
Q34Your dataset has a Date column stored as text. How do you convert it so you can extract the month?
Answer
df['Date'] = pd.to_datetime(df['Date']), then df['Month'] = df['Date'].dt.month. The .dt accessor gives you year, month, day, weekday, hour, etc.
Q35You find that the Age column has values like 200 and -5. How would you handle these?
Answer
These are out-of-range values. You can replace them with the column mean: df.loc[(df['Age'] < 0) | (df['Age'] > 120), 'Age'] = df['Age'].mean(). Or use .clip(0, 120) to cap values at a valid range.
Q36What does axis=0 vs axis=1 mean in pandas?
Answer
axis=0 means "along rows" (vertically — the default). axis=1 means "along columns" (horizontally). So df.drop('col', axis=1) drops a column, while df.drop(0, axis=0) drops a row.
Q37Why would you standardize (z-score normalize) your data before building a model?
Answer
Some models (like KNN and K-means) use distances. If one feature ranges 0–1000 and another 0–1, the large-scale feature dominates. Standardizing puts all features on the same scale (mean=0, std=1) so they contribute equally.
Q38You standardize your training data with StandardScaler. Should you fit the scaler again on the test data?
Answer
No. Use scaler.fit_transform(X_train) on training data, then scaler.transform(X_test) on test data. Fitting on test data would leak information and give unrealistic results.
Q39Your Revenue column is heavily right-skewed (a few very large values). What transformation can help?
Answer
Apply a log transformation: df['Revenue_log'] = np.log1p(df['Revenue']). log1p means log(1+x), which is safe when values are zero. This compresses large values and makes the distribution more bell-shaped.
Q40You want to create a new column that flags customers who spent more than $500. How?
Answer
df['HighSpender'] = np.where(df['Spending'] > 500, 1, 0). This creates a binary 0/1 column based on a condition.
Q41What is feature engineering? Give a simple example.
Answer
Feature engineering means creating new columns from existing data to help a model. Example: if you have purchases and avg_order_value, you could create total_spend = purchases * avg_order_value. The new feature might be more predictive than either original column alone.
Q42How do you quickly identify which columns are categorical (text) and which are numeric?
Answer
df.select_dtypes(include=['object']).columns for text columns, and df.select_dtypes(include=['number']).columns for numeric ones.
Q43You have a boolean column is_member with True/False values. How do you convert it to 1/0?
Answer
df['is_member'] = df['is_member'].astype(int). True becomes 1, False becomes 0. Make sure there are no NaN values first — they'll cause an error.
Q44You want to fill all missing numeric values across the entire DataFrame at once. How?
Answer
df = df.fillna(df.mean(numeric_only=True)). This fills each numeric column's NaN values with that column's mean. numeric_only=True skips text columns.
Q45How do you check if your numeric columns are skewed?
Answer
df[num_cols].skew() — this returns the skewness for each column. Values close to 0 are roughly symmetric. Absolute skewness greater than 3 indicates strong skew, and you might want to apply a log transform.
Data Selection & Filtering (Q46–Q57)
Q46You want to see only the Name and Salary columns from your DataFrame. How?
Answer
df[['Name', 'Salary']]. Use double brackets [[ ]] to select multiple columns — this returns a DataFrame. Single brackets return a Series.
Q47What is the difference between loc and iloc?
Answer
loc selects by label names (and the end index is inclusive). iloc selects by integer position (and the end index is exclusive, like Python slicing). Example: df.loc[0:2] returns 3 rows; df.iloc[0:2] returns 2 rows.
Q48Your manager asks: "Show me all transactions from Branch A." How do you filter?
Answer
df.loc[df['Branch'] == 'A']. This returns only the rows where the Branch column equals 'A'.
Q49You need transactions from Branch A where quantity is greater than 5. How do you combine two conditions?
Answer
df.loc[(df['Branch'] == 'A') & (df['Qty'] > 5)]. Each condition must be in parentheses. Use & for AND and | for OR — not Python's and/or.
Q50You filter your DataFrame and then try to modify the result, but get a SettingWithCopyWarning. Why?
Answer
The filtered DataFrame shares memory with the original. Add .copy() to create an independent copy: subset = df.loc[df['City'] == 'London'].copy(). Now you can safely modify subset.
Q51Before building a model, you need to separate features (X) from the target (y). How?
Answer
X = df.drop('Target', axis=1) and y = df['Target']. Make sure X does not contain the target column — that would be data leakage.
Q52You want to select all columns that start with "genre_". How?
Answer
genre_cols = [c for c in df.columns if c.startswith('genre_')]. This is a list comprehension that filters column names by prefix.
Q53You have retail data with cancelled orders (invoice numbers starting with "C"). How do you remove them?
Answer
df = df[~df['InvoiceNo'].astype(str).str.startswith('C')]. The ~ is the NOT operator — it keeps rows that do NOT start with 'C'.
Q54What does the ~ operator do in a pandas filter?
Answer
It means NOT — it inverts the boolean condition. df[~condition] returns rows where the condition is False. Useful for excluding rows that match a pattern.
Q55You want to check if a text column contains the word "sale" (case-insensitive). How?
Answer
df[df['Description'].str.contains('sale', case=False, na=False)]. case=False makes it case-insensitive. na=False treats NaN values as False instead of raising an error.
Q56You want to compare Member vs Normal customers. How do you create two separate DataFrames?
Answer
members = df.loc[df['Customer type'] == 'Member'] and normals = df.loc[df['Customer type'] == 'Normal']. Then compare using .describe() on each.
Q57What happens if you write df['Branch'] = 'A' instead of df['Branch'] == 'A'?
Answer
= is assignment — it would overwrite the entire Branch column with the value 'A'. == is comparison — it returns True/False for each row. This is a very common beginner mistake.
Aggregation & Grouping (Q58–Q68)
Q58Your manager asks: "How many customers are in each city?" What do you use?
Answer
df['City'].value_counts(). This counts how many times each city appears, sorted from most to least frequent.
Q59Instead of raw counts, you want to see what percentage of customers are in each city. How?
Answer
df['City'].value_counts(normalize=True). Adding normalize=True converts counts to proportions that sum to 1.0.
Q60What is the difference between value_counts() and nunique()?
Answer
value_counts() gives you a full breakdown — how many times each value appears. nunique() gives you just one number — how many distinct values exist. Think of it as the detailed list vs the summary count.
Q61You want to compute the average and maximum sales for each product category. How?
Answer
df.groupby('Category')['Sales'].agg(['mean', 'max']). The agg() method lets you apply multiple functions at once.
Q62What is the difference between groupby().agg() and groupby().transform()?
Answer
agg() collapses each group into one summary row (fewer rows). transform() broadcasts the group result back to every original row (same number of rows). Use transform when you want to add a group statistic as a new column.
Q63You want to add a column showing how many songs each artist has in the dataset. How?
Answer
df['artist_songs'] = df.groupby('artist')['track_id'].transform('count'). Transform broadcasts the count back to every row for that artist.
Q64You want to see the min and max of Age and Purchases side by side. How?
Answer
df[['Age', 'Purchases']].agg(['min', 'max']). This produces a table with min and max for each column.
Q65After clustering your data, you want to see the average feature values per cluster. How?
Answer
df.groupby('cluster')[features].mean().round(2). This creates a "cluster profile" showing what makes each cluster different.
Q66You want to compare the customer type distribution between Branch A and Branch B. How?
Answer
Compute value_counts(normalize=True) for each branch separately, then combine with pd.concat([sA, sB], axis=1). This gives you a side-by-side comparison of proportions.
Q67What does groupby().cumcount() do?
Answer
It gives a running count within each group, starting from 0. For example, if an artist has 5 songs, they get numbered 0, 1, 2, 3, 4. Useful for creating a "song number" or "visit number" column.
Q68You run value_counts() but NaN values are not shown. How do you include them?
Answer
df['col'].value_counts(dropna=False). By default, NaN is excluded. Adding dropna=False includes it in the count.
Visualization (Q69–Q80)
Q69You want to quickly see the distribution of every numeric column in your dataset. What's the fastest way?
Answer
df.hist(figsize=(20, 25)) followed by plt.show(). This creates one histogram per numeric column in a grid layout.
Q70Your manager asks for a bar chart showing how many customers are Male vs Female. How?
Answer
df['Gender'].value_counts().plot(kind='bar'). Use kind='barh' for horizontal bars if labels are long.
Q71You want to see if there's a relationship between Quantity and Gross Income. What chart do you use?
Answer
A scatter plot: plt.scatter(df['Quantity'], df['gross income']). Scatter plots show relationships between two numeric variables. But remember — correlation does not prove causation.
Q72What does sns.regplot() add compared to a regular scatter plot?
Answer
It adds a linear regression trend line and a 95% confidence band. This helps you visually assess whether there's a linear relationship and how strong it is.
Q73You want to see which numeric columns are correlated with each other. What visualization do you use?
Answer
A correlation heatmap: sns.heatmap(df.corr(numeric_only=True), annot=True, cmap='coolwarm'). Values close to +1 or -1 indicate strong correlation. annot=True prints the numbers on the heatmap.
Q74After building a classifier, you want to see where it makes mistakes. What do you plot?
Answer
A confusion matrix: ConfusionMatrixDisplay.from_predictions(y_test, y_pred). It shows how many predictions were correct (diagonal) and where the model confused one class for another (off-diagonal).
Q75What does an ROC curve show, and what does AUC mean?
Answer
An ROC curve plots the True Positive Rate vs False Positive Rate at different thresholds. AUC (Area Under the Curve) summarizes it: 1.0 = perfect, 0.5 = random guessing. Higher AUC = better at ranking positives above negatives.
Q76You built a regression model and want to see how close predictions are to actual values. What do you plot?
Answer
A predicted vs actual scatter plot. Points on the diagonal line mean perfect predictions. The more spread out the points, the larger the errors. Use plt.scatter(y_test, y_pred) and draw a diagonal reference line.
Q77You're plotting training error and test error as you increase model complexity. What pattern indicates overfitting?
Answer
When training error keeps going down but test error starts going up — the gap between them widens. This means the model is memorizing training data instead of learning general patterns.
Q78What is the "elbow method" in K-means clustering?
Answer
Plot the sum of squared distances (SSD/inertia) for different values of K. The SSD always decreases as K increases. Look for the "elbow" — the point where adding more clusters gives diminishing returns. That's your optimal K.
Q79You want to color scatter plot points by their cluster assignment. How?
Answer
sns.scatterplot(data=df, x='feature1', y='feature2', hue='cluster'). The hue= parameter colors each point by its cluster label.
Q80After training a decision tree, you want to see which features matter most. How do you visualize this?
Answer
Plot feature importances as a bar chart: create a Series from model.feature_importances_ with column names as index, sort it, and call .plot(kind='bar'). Higher bars = more important for the tree's decisions.
Supervised vs Unsupervised & General ML Concepts (Q81–Q95)
Q81Your manager asks: "What's the difference between supervised and unsupervised learning?" How do you explain it simply?
Answer
Supervised learning has a target variable (something you're trying to predict) — like predicting whether a customer will buy. Unsupervised learning has no target — it discovers patterns on its own, like grouping similar customers together.
Q82Which of these are supervised: Decision Trees, K-Means, Logistic Regression, Association Rules, KNN?
Answer
Supervised: Decision Trees, Logistic Regression, KNN (they all need a target variable to learn from). Unsupervised: K-Means (clustering) and Association Rules (finding item co-occurrences).
Q83Why do we split data into training and testing sets?
Answer
To estimate how well the model will perform on new, unseen data. If you evaluate on the same data you trained on, you get an overly optimistic score. The test set simulates "real world" data the model has never seen.
Q84What does test_size=0.2 mean in train_test_split()?
Answer
It means 20% of the data goes to the test set and 80% to the training set. A common split. random_state=42 ensures you get the same split every time you run the code.
Q85What does stratify=y do in train_test_split()?
Answer
It ensures both the training and test sets have the same proportion of each class. Essential when classes are imbalanced — for example, if only 10% of customers churned, stratify ensures both sets have roughly 10% churners.
Q86What is overfitting, and how can you tell it's happening?
Answer
Overfitting means the model memorized the training data (including noise) instead of learning general patterns. Sign: very high accuracy on training data but much lower accuracy on test data. The gap between train and test performance is the red flag.
Q87What is cross-validation, and why is it better than a single train/test split?
Answer
Cross-validation (e.g., 5-fold) splits data into 5 parts, trains on 4 and tests on 1, rotating 5 times. Every data point gets tested once. It gives a more reliable performance estimate than a single random split.
Q88What is the difference between predict() and predict_proba()?
Answer
predict() gives you the final class label (e.g., 0 or 1). predict_proba() gives you the probability for each class (e.g., 30% chance of class 0, 70% chance of class 1). You need probabilities for ROC curves and AUC.
Q89Your model has 95% accuracy but your dataset is 95% "No" and 5% "Yes". Is the model good?
Answer
Not necessarily. A model that always predicts "No" would also get 95% accuracy. With imbalanced data, look at precision, recall, F1-score, and AUC instead of just accuracy.
Q90In a confusion matrix, what are True Positives, False Positives, True Negatives, and False Negatives?
Answer
TP = correctly predicted positive. FP = predicted positive but actually negative (false alarm). TN = correctly predicted negative. FN = predicted negative but actually positive (missed case). Rows = actual, columns = predicted.
Q91What is precision, and when does it matter most?
Answer
Precision = TP / (TP + FP) — of all the items you predicted as positive, how many actually were? It matters when false positives are costly. Example: flagging legitimate transactions as fraud (annoying customers).
Q92What is recall, and when does it matter most?
Answer
Recall = TP / (TP + FN) — of all the actual positives, how many did you catch? It matters when false negatives are costly. Example: missing a disease diagnosis (patient doesn't get treatment).
Q93What is the F1 score?
Answer
F1 is the harmonic mean of precision and recall. It balances both metrics into a single number. F1 = 1.0 is perfect. Use it when you care about both false positives and false negatives equally.
Q94What is the difference between MAE and RMSE for regression?
Answer
MAE (Mean Absolute Error) treats all errors equally. RMSE (Root Mean Squared Error) penalizes large errors more because it squares them first. If a few big mistakes are worse than many small ones, RMSE is more informative.
Q95What does R-squared (R²) tell you about a regression model?
Answer
R² measures how much of the variance in the target your model explains. R²=1 means perfect predictions. R²=0 means the model is no better than just predicting the average. R² can even be negative if the model is worse than the mean.
Decision Trees (Q96–Q108)
Q96In simple terms, how does a decision tree make predictions?
Answer
It asks a series of yes/no questions about the features (e.g., "Is Age > 42?"), splitting the data at each step. It keeps splitting until it reaches a final prediction (a leaf node). Think of it like a flowchart.
Q97What is the difference between a Decision Tree Classifier and a Decision Tree Regressor?
Answer
A Classifier predicts categories (e.g., Yes/No, churned/not churned) and is evaluated with accuracy, F1, AUC. A Regressor predicts continuous numbers (e.g., price, revenue) and is evaluated with MAE, RMSE, R².
Q98What does criterion='entropy' vs criterion='gini' mean when training a decision tree?
Answer
Both measure how "impure" a node is. Entropy uses information gain; Gini uses Gini impurity. They usually produce similar trees. Gini is the default and slightly faster to compute.
Q99Your decision tree has 500 nodes and gets 99% training accuracy but only 70% test accuracy. What's happening?
Answer
The tree is overfitting — it's too complex and has memorized the training data. You need to simplify it using pruning: set max_depth, increase min_samples_leaf, or increase min_samples_split.
Q100What is pre-pruning in decision trees? Name three parameters you can set.
Answer
Pre-pruning limits the tree's growth during training. Key parameters: max_depth (maximum tree depth), min_samples_split (minimum samples to split a node), min_samples_leaf (minimum samples in a leaf). Smaller depth and larger min_samples = simpler tree.
Q101How do you read a decision tree node when you visualize it?
Answer
Each node shows: (1) the split rule (e.g., Age ≤ 42.5), (2) samples = percentage of data reaching this node, (3) value = class distribution, (4) class = the majority prediction. If proportion=True, value shows proportions instead of counts.
Q102What does feature_importances_ tell you after training a decision tree?
Answer
It shows how much each feature contributed to the tree's decisions. Values sum to 1.0. Higher importance = the feature was used more often and effectively for splitting. It helps you understand which variables matter most.
Q103What is GridSearchCV, and why would you use it with a decision tree?
Answer
GridSearchCV tries every combination of hyperparameters you specify (e.g., max_depth=[3,5,8], min_samples_leaf=[1,5,10]) using cross-validation. It finds the best combination automatically instead of you guessing.
Q104After running GridSearchCV, how do you get the best model?
Answer
grid.best_estimator_ gives you the model with the best parameters, already refit on the full training data. grid.best_params_ shows which parameter values won.
Q105Should you run GridSearchCV on the full dataset or just the training set?
Answer
Only on the training set. The test set should be used once at the very end to evaluate the best model. If you include test data in tuning, you're leaking information and your results won't reflect real-world performance.
Q106You want to compare how a decision tree performs at different max_depth values. How do you do this manually?
Answer
Loop through depth values, train a tree at each depth, and record both training and test accuracy. Then plot them. Where test accuracy peaks (before it starts dropping) is the sweet spot.
Q107What does classification_report() show you?
Answer
It prints precision, recall, and F1-score for each class, plus macro and weighted averages. It's a quick one-stop summary of how well your classifier performs on every class.
Q108You trained a decision tree and want to check how many nodes and how deep it is. How?
Answer
dt.tree_.node_count for the number of nodes and dt.tree_.max_depth for the depth. These are accessed via the .tree_ attribute of the fitted model.
Linear & Logistic Regression (Q109–Q118)
Q109In plain English, what does linear regression do?
Answer
It draws the best-fit straight line through your data to predict a continuous number. For example, predicting house price based on square footage. It minimizes the sum of squared errors between predictions and actual values.
Q110What is the difference between linear regression and logistic regression?
Answer
Linear regression predicts a continuous number (e.g., price, revenue). Logistic regression predicts a probability of belonging to a class (e.g., will the customer buy? Yes/No). Despite the name, logistic regression is a classification method.
Q111After training a linear regression model, how do you interpret the coefficients?
Answer
Each coefficient tells you how much the prediction changes when that feature increases by 1 unit (holding others constant). Positive coefficient = increases prediction. Negative = decreases. But coefficients are only comparable if features are standardized.
Q112Does linear regression have hyperparameters to tune?
Answer
Basic linear regression (OLS) has no hyperparameters — it just finds the line that minimizes squared error. That's one of its advantages: simplicity. Regularized versions (Ridge, Lasso) do have a tuning parameter.
Q113You train a logistic regression model and get a convergence warning. What should you do?
Answer
Increase max_iter: LogisticRegression(max_iter=5000). The default (100) may not be enough for the algorithm to find the optimal solution, especially with many features.
Q114How does logistic regression decide whether to predict class 0 or class 1?
Answer
It computes a probability. If the probability of class 1 is ≥ 0.5, it predicts 1; otherwise 0. You can get the actual probabilities with model.predict_proba(X)[:, 1].
Q115You want to compare the coefficients of a logistic regression model. Why should you standardize features first?
Answer
If features are on different scales (e.g., age 0–100 vs income 0–100000), the coefficients reflect the scale, not the importance. After standardization, all features have mean=0 and std=1, so coefficients become directly comparable.
Q116You're predicting house prices. Should you use linear regression or logistic regression?
Answer
Linear regression — because house price is a continuous number. Logistic regression is for classification (predicting categories like Yes/No). Use MAE, RMSE, and R² to evaluate the linear regression model.
Q117You're predicting whether a customer will accept a personal loan offer. Which model?
Answer
Logistic regression (or any classifier like decision tree, KNN, naive bayes) — because the target is binary (Yes/No). Evaluate with accuracy, precision, recall, F1, and AUC.
Q118Can linear regression capture non-linear relationships?
Answer
By itself, no — it fits a straight line. But you can add polynomial or interaction features (e.g., x², x1 * x2) to capture curves. Linear regression is "linear in parameters" — the features themselves can be transformed.
Naive Bayes (Q119–Q124)
Q119In simple terms, how does Naive Bayes work?
Answer
It learns the distribution of each feature for each class (e.g., what does "Income" look like for customers who bought vs didn't buy). For a new data point, it combines the evidence from all features to compute the probability of each class. The "naive" part: it assumes features are independent of each other.
Q120Why is it called "naive"?
Answer
Because it assumes all features are independent given the class — meaning knowing one feature tells you nothing about another. This is rarely true in real data (e.g., income and education are correlated), but the model often works surprisingly well anyway.
Q121Should you standardize or transform features before using Gaussian Naive Bayes?
Answer
Yes. Gaussian Naive Bayes assumes features follow a bell-shaped (normal) distribution. If your data is heavily skewed, apply a log transform (np.log1p()) first. Standardization also helps.
Q122How do you interpret which features are most important in a Naive Bayes model?
Answer
Compare the mean of each feature between classes: df.groupby('target')[features].mean(). Features where the means differ a lot between classes are the most discriminating — they help the model tell classes apart.
Q123What does nb.class_prior_ show after training a Naive Bayes model?
Answer
It shows the prior probability of each class based on the training data. For example, if 70% of training samples are class 0 and 30% are class 1, the priors are [0.7, 0.3]. The model uses these as a starting point before looking at features.
Q124What is the difference between Naive Bayes and Logistic Regression?
Answer
Naive Bayes learns the distribution of features per class and combines them (generative). Logistic Regression directly models the decision boundary between classes (discriminative). NB assumes feature independence; LR does not. NB can work well with small data; LR often better with large data.
Association Rules (Q125–Q135)
Q125A supermarket wants to know which products are frequently bought together. What technique do you use?
Answer
Association rule mining. You find frequent itemsets (products that appear together often) and then generate rules like "customers who buy bread and butter also tend to buy milk." This is unsupervised — no target variable.
Q126What do support, confidence, and lift mean in association rules?
Answer
Support = how often the itemset appears in all transactions. Confidence = P(B|A) — if someone buys A, how likely they also buy B. Lift = observed co-occurrence / expected if independent. Lift > 1 means positive association; lift = 1 means independent.
Q127A rule has high confidence (0.9) but lift close to 1.0. Should you trust it?
Answer
Be cautious. High confidence can be misleading if the consequent (the "then" item) is very popular. Lift close to 1 means the items are essentially independent — buying A doesn't really increase the chance of buying B. Always check lift.
Q128What is the Apriori algorithm, and what is its key insight?
Answer
Apriori finds frequent itemsets efficiently. Its key insight: if an itemset is NOT frequent, then no superset of it can be frequent either. This lets it prune the search space dramatically instead of checking every possible combination.
Q129Before running the Apriori algorithm, what format does your data need to be in?
Answer
A boolean (True/False) or binary (0/1) "basket matrix" where rows are transactions and columns are products. A 1 means the product was in that transaction. Use TransactionEncoder or groupby().unstack() to create it.
Q130What does min_support=0.02 mean when running Apriori?
Answer
Only itemsets that appear in at least 2% of all transactions will be kept. Too low = slow computation and too many results. Too high = you miss less common but interesting patterns. It's a trade-off.
Q131You generated association rules and want to filter for meaningful ones. What conditions do you apply?
Answer
Filter for lift > 1 (positive association) and leverage > 0 (co-occurrence exceeds expectation). You can also set a minimum confidence threshold. Example: rules[(rules['lift'] > 1) & (rules['leverage'] > 0)].
Q132A customer has items A and B in their cart. How do you find product recommendations using association rules?
Answer
Filter rules where the antecedent is a subset of the cart: cart = frozenset(['A', 'B']), then rules[rules['antecedents'].apply(lambda x: x.issubset(cart))]. Sort by lift to get the strongest recommendations.
Q133Why are antecedents and consequents stored as frozenset objects?
Answer
Because itemsets can contain multiple items and need to be hashable (usable as dictionary keys or in sets). A frozenset is an immutable set. To query a single item, use frozenset(['item_name']).
Q134How would you manually calculate the confidence of the rule {A} → {B}?
Answer
Confidence = Support(A and B) / Support(A). Count how many transactions contain both A and B, divide by how many contain A. Example: if A appears in 100 transactions and A+B appear together in 40, confidence = 40/100 = 0.4.
Q135Is association rule mining supervised or unsupervised? Why?
Answer
Unsupervised. There is no target variable to predict. You're simply counting co-occurrences in transaction data to discover patterns. No training/test split is needed.
K-Means Clustering & KNN (Q136–Q150)
Q136Your marketing team wants to segment customers into groups based on their behavior. What technique do you use?
Answer
K-Means clustering. It's an unsupervised method that groups similar data points into K clusters. You don't need a target variable — the algorithm discovers the groups on its own based on feature similarity.
Q137Why must you standardize features before running K-Means?
Answer
K-Means uses distances to assign points to clusters. If one feature ranges 0–100000 (income) and another 0–10 (rating), income will dominate all distance calculations. Standardizing puts all features on the same scale.
Q138How do you choose the right number of clusters (K) in K-Means?
Answer
Use the elbow method: run K-Means for K=1 to 20, plot the SSD (inertia) for each K. Look for the "elbow" — the point where adding more clusters gives diminishing returns. Also consider whether the clusters make business sense.
Q139What does inertia_ measure in K-Means?
Answer
It's the sum of squared distances from each point to its assigned cluster center. Lower inertia = tighter clusters. It always decreases as K increases (more clusters = closer centers), which is why you look for the elbow, not the minimum.
Q140What does n_init=10 mean in K-Means?
Answer
K-Means starts with random cluster centers, so different runs can give different results. n_init=10 runs the algorithm 10 times with different starting points and keeps the best result (lowest inertia).
Q141After clustering, how do you understand what each cluster represents?
Answer
Create a cluster profile: df.groupby('cluster')[features].mean(). This shows the average feature values per cluster. For example, Cluster 0 might have high income and low age — "young professionals." Give each cluster a meaningful business label.
Q142What is the key difference between K-Means and KNN?
Answer
K-Means is unsupervised clustering (no target variable, discovers groups). KNN is supervised prediction (needs labeled data, predicts based on nearest neighbors). They both use distances, but for completely different purposes.
Q143How does KNN (K-Nearest Neighbors) make a prediction?
Answer
For a new data point, it finds the K closest points in the training data. For classification, it predicts the majority class among those K neighbors. For regression, it predicts the average of their target values.
Q144Why must you standardize features before using KNN?
Answer
Same reason as K-Means — KNN uses distances. Without standardization, features with larger scales dominate the distance calculation, and the model effectively ignores small-scale features.
Q145What happens if you set K=1 in KNN? What about a very large K?
Answer
K=1 means the prediction is based on the single closest neighbor — very sensitive to noise (overfitting). Very large K smooths out predictions too much (underfitting). You need to find the sweet spot, often by plotting accuracy vs K.
Q146What is the difference between KNeighborsClassifier and KNeighborsRegressor?
Answer
Classifier predicts the majority class among K neighbors (for categorical targets). Regressor predicts the average target value of K neighbors (for continuous targets). Same algorithm, different output.
Q147You want to compare Decision Tree, Logistic Regression, and Naive Bayes on the same dataset. What's the fair way to do it?
Answer
Use the same train/test split for all models. Train each, predict on the same test set, and compare metrics (accuracy, F1, AUC). Using different splits would make the comparison unfair.
Q148A colleague says "the hardest part of machine learning is choosing the right algorithm." Do you agree?
Answer
Not really. The hardest part is usually data cleaning and feature engineering — getting the data into good shape. Most algorithms follow the same fit/predict pattern and are easy to swap. Good features matter more than the perfect algorithm.
Q149What does cross_val_predict() give you that cross_validate() does not?
Answer
cross_validate() returns metric scores per fold (e.g., accuracy for each of 5 folds). cross_val_predict() returns actual predictions for every sample — each predicted by a model that did NOT train on it. Useful for building ROC curves from CV.
Q150You've built a model and your manager asks: "Can we use this in production?" What should you check first?
Answer
Check test set performance (not training performance). Look at precision, recall, and F1 — not just accuracy. Make sure the model was evaluated on data it never saw during training or tuning. Also consider: is the test data representative of real-world conditions? If performance is good and the data is representative, it's a reasonable candidate.
